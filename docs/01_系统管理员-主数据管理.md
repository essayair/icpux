# 矿产供应链运营管理系统
## 01 系统管理员 · 主数据管理

---

## 角色定位

系统管理员负责全平台基础档案的配置与维护。本角色是系统正式运行的**前提条件**，所有业务模块（采销、物流、库存、质检、地磅等）的下拉选项、权限控制、数据口径均依赖主数据的准确配置。

> **在整体业务链中的位置**：主数据管理无上游业务依赖，是所有业务流的"地基"。配置质量直接影响全链路数据准确性、权限安全性与统计口径一致性。

---


## 业务链位置

```
【主数据管理】
       ↓ 支撑
采销业务员 · 物流运营 · 仓储库管 · 现场一线 · 财务结算 · 运营管理者
```

**上游任务流**：无业务前置依赖，由系统管理员在系统上线前完成初始配置，并在业务运营中持续维护更新。

**下游影响**：
- 采销业务员：引用供应商/客户/物料/价格创建合同
- 物流运营：引用车辆、承运商信息创建运单
- 仓储库管：引用仓库库位信息管理库存
- 财务结算：引用组织架构（部门组）进行资金计息与统计
- 全体角色：用户权限决定每个人的菜单可见范围和数据访问边界

## 核心功能模块

### 1. 供应商 / 客户管理
**优先级：P0**

| 功能 | 说明 |
|------|------|
| 基本信息维护 | 企业名称、统一社会信用代码、联系方式、开票信息 |
| 分类分级 | 核心 / 普通 / 临时；供应商与客户分类型管理 |
| 多主体绑定 | 支持一个往来企业绑定多个我方签约主体 |
| 编码唯一性 | 供应商/客户编码系统自动生成且唯一，防止重复录入 |
| 合作状态 | 有效 / 停用，停用后不可在新单据中引用 |

> **运维注意**：供应商/客户档案变更（如开票信息、联系方式）应及时更新，已生效历史单据不受影响，新单据引用最新版本。

---

### 2. 物料管理
**优先级：P0**

| 功能 | 说明 |
|------|------|
| 基础信息 | 品名、规格（萤石粉等级/粒度）、大类、计量单位 |
| 商品编码 | 支持内部编码与外部商品编码（税务/海关）转化映射 |
| 质检指标联动 | 物料与质检指标模板关联，入库时自动触发对应质检项 |
| 虚拟物料 | 支持加工转化场景下的虚拟物料（如委托加工成品） |

> **运维注意**：物料新增或规格调整需同步评估质检指标模板是否需要更新（关联质检化验模块）。

---

### 3. 仓库库位管理
**优先级：P0**

| 功能 | 说明 |
|------|------|
| 仓库基本信息 | 仓库名称、地址、类型（自有仓/监管仓/第三方仓） |
| 库区/库位编码 | 支持多级层级：仓库 → 库区 → 库位 |
| 库容配置 | 设置库位额定库容，用于库存预警 |
| 地磅点位关联 | 仓库与地磅设备点位关联，入库时自动匹配过磅数据 |

---

### 4. 车辆管理
**优先级：P0**

| 功能 | 说明 |
|------|------|
| 车辆档案 | 车牌号、车型、额定载重、所属物流供应商 |
| 驾驶员信息 | 姓名、联系方式、驾驶证信息 |
| 北斗/GPS设备 | 绑定北斗设备号，用于在途轨迹跟踪 |
| 物流运单联动 | 车辆信息与物流运单、地磅过磅自动关联 |

---

### 5. 价格管理
**优先级：P0**

| 功能 | 说明 |
|------|------|
| 基础价格维护 | 按物料维护基础参考价格与生效期 |
| 历史价格 | 记录历史价格变化，支持回溯查看 |
| 区域价格差异 | 支持按产地/区域配置差异化价格基准 |
| 采销合同引用 | 合同定价时引用价格基准作为参考，实际价格由合同约定 |

---

### 6. 组织架构管理
**优先级：P1**

| 功能 | 说明 |
|------|------|
| 部门组 | 用于额度、资金计息、权限的多维度统计口径 |
| 部门 / 业务组 | 归属业务部门，用于报表统计 |
| 利润中心 | 内部利润中心管理，支持移库核算场景 |
| 树形层级 | 支持树形组织架构，无限层级 |

---

### 7. 用户与权限管理
**优先级：P1**

| 功能 | 说明 |
|------|------|
| 用户账号管理 | 创建/停用系统用户，关联员工信息 |
| 角色配置 | 按岗位定义角色（采销业务员/物流运营/财务/现场一线等） |
| 菜单/按钮权限 | 控制各角色可见菜单与可操作按钮 |
| 数据权限 | 按部门组维度控制数据可见性（如仅看本部门合同） |
| 业务线隔离 | 合作伙伴共管场景下，按主体关系授权数据访问 |

> **重要原则**：权限配置需严格按最小权限原则配置，敏感数据（结算金额、资金信息）仅对有需要的角色开放。

---

### 8. 审批流程配置
**优先级：P1**

| 功能 | 说明 |
|------|------|
| 审批链配置 | 按单据类型（合同/付款申请/入库单等）配置审批节点 |
| 部门维度 | 支持按部门/金额段配置不同审批链 |
| 审批人设置 | 支持固定审批人和按岗位动态指定 |
| 抄送配置 | 关键单据审批时自动抄送相关人员 |

---

### 9. 基础智能化工具（OCR）
**优先级：P2**

| 功能 | 说明 |
|------|------|
| 单据OCR识别 | 支持磅单、发票、合同、质检报告等单据图像识别 |
| 结构化提取 | 识别结果自动填充至系统字段，减少人工录入 |
| 人工校验 | 识别结果需人工确认后生效，防止错误数据入库 |

---

## 配置优先顺序（系统上线前）

**第一批（上线前必须就绪）**：
1. 供应商/客户档案 - 采销合同无法创建
2. 物料基础信息 - 所有业务单据依赖
3. 仓库库位 - 库存管理无法运行
4. 车辆档案 - 物流运单无法填写
5. 价格基准 - 合同定价参考

**第二批（上线同期配置）**：
6. 组织架构与部门 - 权限和统计依赖
7. 用户账号与角色权限 - 各岗位人员需要账号登录
8. 审批流程 - 合同/付款等关键单据的审批链

**第三批（稳定运行后接入）**：
9. OCR识别工具 - 提效工具，非核心流程

---

## 注意事项

- 主数据变更须记录版本历史，**已生效单据不随主数据变更而改变**，新单据始终引用最新版本
- 供应商/客户停用前，需确认无未完成的合同执行中
- 组织架构调整（如新增部门组）需同步评估对资金计息规则和权限范围的影响
- 未来引入第三方信用管理、风控模块时，供应商/客户档案将作为底层数据扩展点
