# 矿产供应链运营管理系统
## 02 采销业务员 · 合同管理

---

## 角色定位

采销业务员是**业务链的起点**，负责与上下游洽谈贸易条款并在系统中录入合同，驱动后续物流、库存、结算全流程。合同一旦生效，将成为下游所有单据（发货指令、物流运单、入库单、结算单）的业务依据。

> **典型岗位**：业务经理、采购对接、销售内勤（如黄蕾、李纯福、蔡宇豪）

---

## 业务链位置

```
【主数据管理】（供应商/客户/物料/价格）
       ↓ 引用
【采销业务员 · 合同管理】← 你在这里
  ↙                    ↘
物流运营                财务结算
（发货指令→派车）        （结算单→付款）
  ↓
仓储库管
（入库/出库）
```

---

## 上游任务流：开始操作前需确认

| 前置条件 | 来源 | 说明 |
|---------|------|------|
| 供应商/客户档案已建立 | 系统管理员 | 合同必须从主数据选择对手方，不可手动录入 |
| 物料信息已配置 | 系统管理员 | 合同货物信息依赖物料主数据 |
| 价格基准已维护 | 系统管理员 | 为合同定价提供参考依据 |
| 额度已设置（如需） | 系统管理员 | 超额时系统预警或拦截 |

## 核心功能模块

### 合同类型总览

| 合同类型 | 适用场景 | 优先级 |
|---------|---------|--------|
| 采购合同 | 向上游供应商采购货物 | P0 |
| 销售合同 | 向下游客户销售货物 | P0 |
| 双边合同 | 以销定购，同时确定采销两端 | P0 |
| 框架协议（年度/月度） | 长期合作，约定总量/总价框架 | P1 |
| 采购意向单 | 合同审批中/未录入时先行记录 | P1 |
| 销售意向单 | 销售意向先行确认 | P1 |
| 委托加工合同 | 委托第三方加工生产 | P2 |

### 一、采购合同

#### 1.1 合同操作功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 创建合同 | 支持手动录入、引用采购意向单创建、导入Excel批量创建、复制已有合同创建；创建时校验供应商主数据、额度、价格 | P0 |
| 查询合同 | 按合同编号、供应商、状态、日期范围、物料、部门等多维度组合查询；支持模糊搜索 | P0 |
| 修改合同 | 仅"已保存""已驳回"状态可修改；修改后重新进入审批流程；保留历史版本记录 | P0 |
| 关闭/作废合同 | 执行完毕或不再执行时关闭/作废；关闭时校验溢短装比例范围内的已执行量 | P0 |
| 保存草稿 | 草稿状态保存，离开页面时提示保存 | P1 |
| 合同打印 | 按模板格式化打印合同文本 | P1 |
| 版本管理 | 记录合同每次修改的历史版本，支持版本回溯 | P1 |

#### 1.2 合同状态机

```
已保存 → 审批中 → 已驳回（可修改后重新提交）
                 ↓
              已生效 → 执行中 → 已完成
                              ↓
                            已关闭 / 已作废
```

#### 1.3 关键字段

| 字段分组 | 字段名称 | 说明 |
|---------|---------|------|
| 公司信息 | 我方签约主体 | 此次交易使用的我方主体，支持多主体切换 |
| 公司信息 | 业务部门 / 业务员 | 归属部门与负责人 |
| 供应商信息 | 供应商名称 | 从主数据选择，支持模糊搜索 |
| 供应商信息 | 外部合同号 | 供应商方合同编号，用于对账关联 |
| 货物信息 | 物料/商品 | 从物料主数据选择，含商品编码、品名、规格 |
| 货物信息 | 采购数量 / 单价 / 总额 | 合同约定数量、含税/不含税单价、总金额 |
| 货物信息 | 溢装/短装比例 | 允许实际交货数量偏差范围，用于合同关闭校验 |
| 货物信息 | 定价类型 | 一口价 / 基差定价 / 不定价（后续通过定价单确定） |
| 收货信息 | 收货地/仓库 | 货物送达仓库或地点 |
| 收货信息 | 交货方式 | 供应商配送 / 自提 / 库转等 |
| 结算条款 | 结算数量依据 | 以磅单/入库单/质检单哪个数量为准 |
| 结算条款 | 付款比例 | 预付款比例、尾款比例 |
| 结算条款 | 付款账期 | 约定付款账期天数 |
| 结算条款 | 付款方式 | 电汇/承兑/信用证等 |

#### 1.4 采购合同生成的下游单据

| 下游单据 | 触发节点 | 说明 |
|---------|---------|------|
| 供应商发货指令 | 采购合同生效后 | 通知供应商发货，跟踪货物在途状态 |
| 定价单 | 合同为不定价/基差定价时 | 后续确定具体价格，关联合同行项目 |
| 付款申请 | 满足付款条件时 | 基于合同结算条款生成 |

### 二、销售合同

#### 2.1 合同操作功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 创建合同 | 支持手动录入、引用销售意向单创建、导入Excel批量创建、复制已有合同创建 | P0 |
| 查询合同 | 按合同编号、客户、状态、日期范围、物料、部门等多维度查询 | P0 |
| 修改合同 | 仅"已保存""已驳回"状态可修改；修改后重新审批；保留历史版本 | P0 |
| 关闭/作废合同 | 执行完毕或异常终止时操作；关闭后不可生成后续单据 | P0 |

#### 2.2 销售合同与采购合同的核心差异字段

| 字段 | 说明 |
|------|------|
| 客户名称 | 下游客户（采购合同为供应商） |
| 实际收货方 | 第三方销售场景下实际接收货物的下游方 |
| 交货地点 | 货物交付给客户的地点 |
| 收款条件 | 预收款/货到收款/账期收款等 |
| 逾期条款 | 逾期收款的罚则 |
| 赊销/赊销金额 | 是否赊销及赊销额度，用于风控 |

#### 2.3 销售合同生成的下游单据

| 下游单据 | 触发节点 | 说明 |
|---------|---------|------|
| 销售提单/发货通知单 | 销售合同生效后 | 向下游客户发出提货/发货通知，关联物流出库单 |

**销售合同执行主链路**：
```
销售合同生效
    → 销售提单/发货通知单
    → 物流出库单
    → 过磅出库
    → 库存扣减
    → 客户收货确认
    → 触发结算流程
```

### 三、双边合同（以销定购）

双边合同是"以销定购"业务模式。在同一合同内同时约定上游采购和下游销售，系统自动拆分为独立的采购单与销售单。

#### 3.1 核心流程

```
签订双边合同
    → 系统自动拆分：采购合同 + 销售合同
    → 采购端：生成供应商发货指令（按销售需求控制采购节奏）
    → 销售端：生成销售提单
    → 以销定购：按销售需求驱动采购执行
```

#### 3.2 功能特性

| 特性 | 说明 | 优先级 |
|------|------|--------|
| 自动拆分 | 签订双边合同后，系统自动拆分为采购合同和销售合同，两边独立管理执行 | P0 |
| 提货计划联动 | 供应商发货指令+销售提单联动，按销售需求控制采购节奏 | P0 |
| 直拨业务 | 供应商直接发货到下游客户，减少中间仓储环节 | P1 |
| 采销匹配 | 采购端与销售端通过贸易关联号自动匹配，端到端跟踪 | P1 |
| 库存敞口管理 | 实时计算采销两端差异，展示库存敞口与潜在盈亏 | P2 |

### 四、其他合同

通用OA审批流程合同，如物流供应商合作合同、技术供应商合同、仓储服务合作合同。走标准审批流程，不驱动业务单据流转。

## 完整业务操作流程

**标准采购合同执行链路**：

```
① 采销员录入合同 → 判断单边/双边 → 双边自动拆出采购单与销售单
② 系统校验额度 → 超额弹窗提示或拦截（一期人工控制，合作伙伴总额提示）
③ 合同审批通过（已生效）
④ 下达供应商发货指令 → 自动通知物流运营与仓储库管
⑤ 收货确认 → 结算引擎按规则算出预付款/尾款 → 生成结算单待财务审核
⑥ 持续跟踪：发货指令在途 → 物流入库 → 业务入库 → 结算 → 发票 → 付款
⑦ 合同执行完毕 → 关闭合同 → 归档
```

## 下游任务流：合同生效后你的工作触发

| 下游角色 | 触发事项 | 你需要做什么 |
|---------|---------|------------|
| 物流运营 | 合同生效后下达发货指令 | 在系统中创建并下达供应商发货指令 |
| 仓储库管 | 货物到达后入库 | 确认收货数据，结算引擎据此计算 |
| 财务结算 | 收货确认后触发结算 | 确认结算数据无误后提交财务审核 |
| 现场一线 | 合同信息下达现场 | 通知甘肃现场质量标准、扣重规则、结算周期等 |

## 模块关联关系

| 关联模块 | 关系说明 |
|---------|---------|
| 主数据管理 | 供应商/客户/物料/价格从主数据引用 |
| 物流管理 | 合同生效后下达发货/提货指令，触发物流任务 |
| 库存管理 | 出入库单据关联合同与批次，跟踪履约进度 |
| 质检化验 | 质检结果影响验收确认与结算扣水扣杂 |
| 地磅管理 | 过磅净重作为结算数量依据 |
| 资金付款管理 | 合同结算条款驱动结算单生成和付款申请 |

## 额度管理说明（一期）

- 系统按往来单位设授信额度
- 下单时自动校验可用额度
- 超额时弹窗提示（**一期不做硬拦截，人工控制**，可查看合作伙伴总额）
- 超额发货需人工审批确认
