# 矿产供应链运营管理系统
## 07 运营管理者 · 工作台

---

## 角色定位

运营管理者是**业务全局的监控者与决策者**，通过工作台聚合各模块的待办事项、业务状态、关键预警，以审批、监控和调度的方式推动整体业务运转。工作台是所有其他角色模块数据汇聚的"控制中心"。

> **典型岗位**：业务负责人、综合内勤（如陈飞祥、黄蕾，以及需要全局视角的管理人员）

---

## 业务链位置

```
采销业务员 · 物流运营 · 现场一线 · 仓储库管 · 财务结算
          ↓（所有模块数据汇聚）
【运营管理者 · 工作台】← 你在这里
          ↓（审批决策、异常处置）
  各模块执行层（推动业务继续流转）
```

工作台本身不产生业务数据，而是**汇聚**各模块数据并**推动**业务的及时流转。

---

## 核心功能模块

### 一、待办事项

聚合当前用户的**全部待审批、待处理任务**，避免在各模块间来回切换查找待办。

| 待办类型 | 来源模块 | 说明 |
|---------|---------|------|
| 待审批合同 | 采销系统 | 采购合同/销售合同/双边合同等待审批 |
| 待审批付款申请 | 资金付款管理 | 付款申请等待管理者审批 |
| 待处理质检熔断 | 质检化验 | 质检熔断事件，需人工介入处置（折价/退货） |
| 待处理磅差异常 | 地磅管理 | 磅差超阈值锁单，等待核实和定责 |
| 待确认收货数据 | 现场/采销 | 现场数据汇总后等待负责人确认"数据无误" |
| 待发起结算 | 采销系统 | 收货确认后，等待启动结算流程 |
| 合同即将到期 | 采销系统 | 合同履约截止日前预警，需跟进执行进度 |
| 额度预警 | 采销系统 | 往来单位额度使用率超阈值，需关注 |
| 逾期付款 | 资金付款管理 | 应付账款超过约定付款日未付 |
| 逾期回款 | 资金付款管理 | 应收账款超过约定收款日未收 |

**操作支持**：
- 按待办类型筛选
- 点击待办直接跳转到对应单据详情
- 批量处理（同类型多条审批）

---

### 二、业务概览

关键业务指标的**图表化实时展示**，帮助管理者快速把握整体运营状态。

| 概览模块 | 展示内容 | 说明 |
|---------|---------|------|
| 合同执行情况 | 本期合同数/执行率/逾期率 | 采购/销售合同分别展示 |
| 在途货物 | 在途车次数/在途货量 | 采购入库和销售发货分别展示 |
| 库存状况 | 各物料当前库存量/库龄分布 | 关注滞销批次 |
| 资金占用 | 各业务线资金占用金额/利息测算 | 按部门组维度汇总 |
| 结算进度 | 本期待结算/已结算金额 | 预付款/尾款分别展示 |
| 质检合格率 | 本期化验合格/不合格/熔断数量及比率 | 识别质量趋势 |

---

### 三、预警通知

系统**主动推送**异常事件和即将到期的任务，无需管理者主动巡检。

| 预警类型 | 触发条件 | 紧急程度 |
|---------|---------|---------|
| 质检熔断 | 货物质检结论为熔断 | 高 |
| 磅差异常 | 进出场磅差超过阈值 | 高 |
| 付款逾期 | 应付账款超最晚付款日 | 高 |
| 回款逾期 | 应收账款超最晚收款日 | 高 |
| 额度预警 | 往来单位可用额度低于阈值 | 中 |
| 合同到期预警 | 合同履约截止日前N天 | 中 |
| 在途超时 | 运输超过预计到达时间N小时 | 中 |
| 设备离线 | 地磅设备离线或数据采集中断 | 中 |
| 库龄预警 | 批次在库超过N天 | 低 |

### 四、快捷入口

常用操作的**一键直达**，减少菜单层级跳转。

| 快捷操作 | 说明 |
|---------|------|
| 新建合同 | 快速进入合同录入页面 |
| 查询物流在途 | 直接查看当前在途货物状态 |
| 录入磅单 | 快速跳转磅单上传页 |
| 发起结算 | 从工作台直接发起结算流程 |
| 查看资金看板 | 快速查看当日资金状态 |
| 我的审批 | 快速进入审批待处理列表 |

---

### 五、消息中心

**全平台通知汇聚**，按类型分类管理，不遗漏任何业务消息。

| 消息类型 | 来源 |
|---------|------|
| 审批消息 | 我发起的单据审批结果；需我审批的申请 |
| 业务提醒 | 合同到期、付款到期、结算发起等提醒 |
| 异常通知 | 质检熔断、磅差异常、在途超时等告警 |
| 系统通知 | 主数据变更、系统维护等系统级通知 |

**支持**：已读/未读状态管理；按类型筛选；历史消息查询。

---

### 六、审批管理（运营管理者核心职责）

运营管理者通常是各类业务单据的**审批节点**，工作台集中展示所有待审批事项。

#### 主要审批类型

| 审批单据 | 审批要点 |
|---------|---------|
| 采购/销售合同 | 交易对手、价格、数量、结算条款是否合理；额度是否充足 |
| 双边合同 | 采销匹配逻辑是否合理；直拨安排是否可行 |
| 付款申请 | 付款依据是否完整；金额是否与结算单一致 |
| 库存调拨 | 调拨原因是否合理；涉及利润中心核算的移库 |
| 报损/报溢 | 差异原因是否合理；是否需要追责 |
| 磅差异常处理 | 偏差原因核实；定责决定 |

#### 审批操作建议

- 审批前先查看关联的原始单据（合同、磅单、质检单、结算单）
- 重大金额付款须核实银行账户信息与对方一致
- 质检熔断审批须与采销业务员确认折价/退货方案后再处理

## 业务全链路监控

工作台的最高价值是提供一笔业务从合同到结算的**端到端跟踪视图**。

**一笔采购合同的完整状态追踪**：

```
合同录入（待审批）
    ↓ 审批通过
合同生效（已生效）
    ↓ 下达发货指令
发货在途（物流跟踪中）
    ↓ 货物到厂
过磅质检（数据确认中）
    ↓ 数据确认
结算发起（待结算）
    ↓ 结算审核
付款审批（待付款）
    ↓ 付款完成
发票管理（待开票/已开票）
    ↓ 全部完成
合同关闭（已归档）
```

每个节点的当前状态在工作台中可一目了然，异常节点有预警标记。

## 实现说明

工作台采用**后端逻辑聚合实现**，不依赖AI Agent：

- 待办事项：后端SQL/API实时查询各模块待办数据，准确可靠
- 业务概览：后端统计计算 + 前端图表可视化渲染
- 预警通知：后端定时任务扫描业务数据 + 消息推送机制
- AI辅助（可选）：提供自然语言查询能力（如"本周合同执行异常汇总"），作为辅助层，非核心功能

## 二期展望：对外服务入口

二期阶段，工作台将扩展为**合作伙伴门户的入口**，让外部合作伙伴也能通过对应视图查看与其相关的业务状态：

| 对外能力 | 说明 |
|---------|------|
| 合作伙伴门户 | Web后台 + 小程序/企业微信，合作伙伴查看自己的单据 |
| 信息主动推送 | 合同状态、物流轨迹、磅单数据、质检结果主动推送给相关方 |
| 数据隔离 | 按签约主体隔离，合作伙伴只看到与自己相关的数据 |
| 资金方穿透查询 | 资金方（物产/银行）可查看确权数据和货物状态，辅助放款风控 |

| 关联模块 | 工作台与该模块的关系 |
|---------|-------------------|

| 地磅管理 | 磅差异常预警；设备离线告警 |
| 财务结算 | 付款审批待办；逾期收付款预警；资金看板数据 |
| 主数据管理 | 权限配置决定工作台数据可见范围 |
