# 矿产供应链运营管理系统
## 04 现场一线 · 地磅管理 / 质检化验 / 监控录像

---

## 角色定位

现场一线岗位是**货物进入仓库前的质量与数量关卡**，也是整个业务链中数据可信性的核心保障。过磅数据是结算数量的原始依据，质检数据是验收与扣款的法定依据，监控影像是防作弊与争议追溯的证明链。

本角色覆盖三个紧密协作的子模块，通常由玉门现场团队协同完成：
- **过磅/检斤岗**：地磅管理（郑学）
- **取样/检测岗**：质检化验（郑小明、徐主任）
- **现场留存**：监控录像（配合地磅和质检操作）

> **典型岗位**：过磅员、取样员、化验员、现场负责人（如郑学、郑小明、徐主任、陈飞祥）

## 核心功能模块

---

### 第一部分：地磅管理

#### 模块定位
采集进出场过磅数据，作为**数量结算的源头依据**。通过设备防作弊与影像关联，保证"磅不可造假"与采购/入库偏差可追溯。

一期方案：电脑Web端将本地磅单数据上传到线上系统（共用地磅设备，不共用本地数据，由现场操作上传）。

#### 功能清单

| 功能 | 说明 |
|------|------|
| 过磅数据采集 | 车辆进出场自动称重；记录毛重/皮重/净重；关联车次、运单、入库单 |
| 磅单管理 | 磅单生成、存储、查询与关联；拍照上传留存 |
| 双磅单采集 | 同时采集供方磅单（供应商/矿山端）与我方现场磅单，用于对账核实 |
| 偏差预警 | 出厂重量 vs 到厂重量比对；偏差超阈值自动锁单并推送异常 |
| 防作弊机制 | 设备离线/人工录入打"非可信数据源"标签；阻断自动流转，转人工复核 |
| 影像关联 | 过磅时自动抓拍车头/车厢/空厢影像，与磅单自动关联 |
| 异常处理 | 磅差超限、设备离线、人工录入等异常场景的标记与处理流程 |

#### 关键操作步骤

```
① 车辆入厂 → 检查车厢（有无加水/夹带）
② 车辆过磅（毛重）→ 系统自动取重 + 抓拍车头/车厢影像
③ 卸货
④ 卸货后检查残留，拍照留存
⑤ 车辆出场过磅（皮重）→ 计算净重
⑥ 磅单生成 → 水印照片/磅单数据实时推送工作群
⑦ 回磅复检（必要时）→ 数据异常时发起复核
```

#### 防作弊关键规则

| 场景 | 处理方式 |
|------|---------|
| 设备正常自动取重 | 磅单标记为"可信数据源"，可自动流转到入库流程 |
| 设备离线/人工录入 | 磅单打"**非可信数据源**"标签，**阻断自动流转**，需人工复核确认后才可继续 |
| 进出场重量偏差超阈值 | 系统自动**锁单**并推送异常给现场负责人，等待人工处理 |
| 人工录入的备降磅单 | 需上级确认标注原因，留存审计记录 |

#### 磅单关联链

```
磅单 → 关联物流运单 → 关联发货指令/提单 → 关联采购/销售合同
     ↓
   关联质检单（同一车次）
     ↓
   关联监控影像（过磅时刻抓拍）
```

### 第二部分：质检化验

#### 模块定位
管理入厂货物的取样、化验、报告与质检规范，提供**质量数据作为验收与结算依据**。核心是确定性：过程的确定性、结果的确定性、异常处理的确定性。

质检数据涉及三方：**我方化验、巨化方验收、第三方检测机构**，系统需分别记录三组化验数据并进行对比。

#### 功能清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 取样管理 | 卸货后规范取样并登记，记录取样时间、取样人、取样方法；3份化验品留样 | P0 |
| 附件上传 | 取样照片、化验单/质检报告照片上传；系统自动OCR识别结构化数据 | P0 |
| 化验任务 | 化验分析任务分配与进度跟踪；分配化验人员/设备 | P0 |
| 化验数据录入 | 化验指标结果录入系统（一期手工录入，可对接设备直采） | P0 |
| 质检报告 | 出具质检报告，支持多行检测指标；按规范自动判定合格/不合格/熔断 | P0 |
| 质检规范管理 | 维护质检指标模板和判定标准；按物料配置指标项和阈值（系统管理员操作） | P2 |
| 指标争议处理 | 处理质检结果争议（熔断场景下人工介入） | P2 |
| 质量预警 | 基于化验数据推送质量趋势预警（二期模型能力） | P2 |

#### 三方化验数据管理

| 化验方 | 数据录入方式 | 说明 |
|--------|------------|------|
| **我方化验** | 化验室手动录入系统 | 我方出具化验结果，与巨化比对 |
| **巨化方验收** | 手动录入或上传扫描件OCR识别 | 出具《质检单》作为验收依据，我方存档核对 |
| **第三方检测机构** | 手动录入或协商让第三方自行上传 | 客观参照依据 |

> **偏差处理规则**：我方化验结果与巨化比对，偏差 >0.3% 立即报告现场负责人（陈飞祥）并留样，等待协商处理。

#### 关键操作步骤

```
① 入库/过磅完成 → 系统自动触发质检单
② 规范取样（保证取样代表性）→ 3份化验品分别留样
③ 测水分、测品位等指标
④ 卸车后检查残留
⑤ 化验数据录入系统（化验单/数据/照片实时推送工作群）
⑥ 系统按质检规范自动判定：合格 / 不合格 / 熔断
⑦ 巨化方出具《质检单》→ 我方系统录入存档
⑧ 质检报告关联到入库单和结算单
```

#### 质检结论处理

| 质检结论 | 后续动作 |
|---------|---------|
| **合格** | 触发业务入库，货物进入库存，可正常结算 |
| **不合格（折价）** | 通知采销业务员与供应商协商折价，双方确认后按折价结算 |
| **熔断（拒收）** | 通知采销业务员与客户协商退货，冻结该批次不可入库 |

#### 质检结果对结算的影响

- **扣水**：实测水分超过合同约定水分基准，按合同扣水公式计算扣重数量
- **扣杂**：实测杂质超标，按合同扣杂公式计算扣重数量
- **结算数量** = 磅单净重 - 扣水数量 - 扣杂数量
- 扣重数量作"结算利息"单独处理（财务结算模块）

> **重要**：化验合格才结算开票，质检结果与结算、发货节点强联动。

### 第三部分：监控录像

#### 模块定位
对过磅、卸货、仓库等关键环节进行**视频监控与存证**，与磅单/单据关联，提供防作弊与争议追溯能力。

一期：视频影像本地硬盘存储，关键影像与磅单关联。切片上云、上链放在二期。

#### 功能清单

| 功能 | 说明 |
|------|------|
| 过磅抓拍 | 过磅时自动抓拍车头/车厢/空厢影像，与磅单自动关联；多角度抓拍 |
| 卸货监控 | 卸货过程录像与关键帧抓拍；卸货后抓拍"空厢"，防夹带/未卸净 |
| 仓库监控 | 仓库关键摄像头画面，堆场变动存留 |
| 影像关联 | 影像+磅单+单据打包，作为收货确认与争议追溯证据 |
| 影像存储 | 关键影像数据上云（一期本地存储）；设置保存期限；支持按单据/日期检索 |
| 查看权限 | 按权限控制影像查看，不同角色对不同业务线影像有不同权限 |
| 实时监控 | 关键点位实时视频流查看；对接摄像头/NVR设备 |

#### 关键操作触发链

```
① 车辆过磅 → 自动触发抓拍 → 影像自动关联磅单
② 卸货 → 录制卸货过程 → 卸货后抓拍空厢（防夹带/未卸净）
③ 影像+磅单+质检报告打包归档 → 作为收货确认与争议追溯证据
```

## 三模块协同操作全流程

```
车辆预计到达 → 现场准备
      ↓
车辆入厂检查（查车厢、有无加水）
      ↓
【地磅】进场过磅（毛重）→ 抓拍车头/车厢 → 【监控】记录影像
      ↓
卸货（卸货过程录制）→ 【监控】拍摄空厢
      ↓
【质检】规范取样 → 化验（测水分/品位等）→ 数据录入
      ↓
【地磅】出场过磅（皮重）→ 计算净重 → 磅单生成
      ↓
磅单+化验数据+影像 实时推送工作群（每日18:00前完成数据汇总）
      ↓
现场负责人核对数据 → 确认无误 → 通知采销发起结算
      ↓
【质检合格】→ 通知仓储库管生成业务入库单
```

## 模块关联关系

| 关联模块 | 关系说明 |
|---------|---------|
| 物流管理 | 关联运单/车辆（地磅）；到货/卸货影像关联车次（监控） |
| 库存管理 | 净重作为入库数量（地磅）；质检结论决定批次是否可用（质检） |
| 采销系统 | 质检结果影响验收与结算；通过提单关联合同 |
| 财务结算 | 磅单净重和质检扣重结果共同决定结算数量 |
| 主数据管理 | 物料、质检规范取自主数据 |

## 异常处理说明

| 异常场景 | 处理方式 |
|---------|---------|
| 磅差超阈值（进出场重量差异过大） | 系统自动锁单，推送异常给现场负责人，等待人工核实和定责 |
| 地磅设备离线 | 人工录入磅单，标记"非可信数据源"，阻断自动流转，转人工复核 |
| 质检偏差 >0.3%（我方vs巨化） | 立即报告陈飞祥，留样，等待双方协商 |
| 质检结论为熔断 | 冻结批次，通知采销协商折价或退货，人工介入处理 |
| 视频设备故障 | 记录设备故障时段，磅单/质检单标注影像缺失，留存书面记录 |
