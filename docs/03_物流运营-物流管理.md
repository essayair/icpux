# 矿产供应链运营管理系统
## 03 物流运营 · 物流管理

---

## 角色定位

物流运营是**货物从供应商到仓库、从仓库到客户的组织执行方**，负责叫车调度、运输任务全程跟踪、入库与出库确认，以及与承运商的运费核算对账。运营平台方是物流的组织方，承运商是被调度和履约的一方。

> **典型岗位**：物流对接、物流台账核对、采购对接（如蔡宇豪、李纯福）

---

## 业务链位置

```
【采销业务员 · 合同管理】
（下达供应商发货指令 / 销售提单）
          ↓
【物流运营 · 物流管理】← 你在这里
     ↙              ↘
地磅管理            仓储库管
（车辆过磅）        （触发入库/出库）
     ↓
财务结算
（运费核算归集）
```

## 上游任务流：开始操作前需确认

| 前置条件 | 来源 | 说明 |
|---------|------|------|
| 供应商发货指令已下达 | 采销业务员 | 采购合同生效后，业务员在系统下达发货指令，你接收并创建运输任务 |
| 销售提单已创建 | 采销业务员 | 销售合同生效后，业务员创建提单，你据此安排出库运输 |
| 车辆档案已维护 | 系统管理员 | 运单需从主数据选择车辆，确保车牌、司机信息完整 |
| 承运商信息已录入 | 系统管理员/承运商管理 | 物流供应商资质、运价协议需提前维护 |

### 一、物流运单管理

#### 1.1 创建运单

| 创建方式 | 说明 |
|---------|------|
| 由供应商发货指令发起 | 采购入库场景，关联采购合同发货指令 |
| 由销售提单发起 | 销售发货场景，关联销售合同提单 |
| 独立创建 | 可独立创建后手动关联采购提货或销售提单 |

支持增删改查；关键操作需短流程审核并抄送相关人员。

#### 1.2 物流运单字段

| 字段分组 | 字段名称 | 说明 |
|---------|---------|------|
| 提货信息 | 提货地点/时间 | 货物提取的地点与计划时间 |
| 提货信息 | 发货指令/提单号 | 关联的业务单据编号 |
| 送达信息 | 送达地点/预计到达时间 | 货物送达目的地与预计时间 |
| 物流供应商 | 物流公司 | 承运的物流公司 |
| 物流供应商 | 司机/车辆/车牌 | 执行运输的司机与车辆信息 |
| 物流供应商 | 里程 | 运输里程数 |
| 运输信息 | 运输方式 | 汽运/铁运/船运等 |
| 运输信息 | 运输状态 | 待发运 / 在途 / 已到达 / 已签收 |
| 运输费用 | 运费/运杂费 | 运输相关费用 |
| 运输费用 | 费用承担方 | 我方/对方承担运输费用 |
| 商品信息 | 物料/数量 | 运输的货物明细 |

#### 1.3 物流查询

对接第三方物流平台，查询货物在途实时状态。在第三方物流系统中获取当前位置、预计到达时间、在途异常信息。

### 二、物流入库单（采购到货确认）

#### 2.1 功能说明

货物过磅进入仓库后，通过物流入库单完成到货确认，触发库存形成流程。

**数据来源**：发货指令 + 物流运单 + 地磅过磅数据

#### 2.2 操作流程

```
方式一：选择发货指令 → 选择物流运单 → 地磅数据自动带入 → 入库确认
方式二：直接选择物流运单 → 手动关联地磅数据 → 入库确认
```

#### 2.3 关联关系

```
物流入库单 → 业务入库单（库存形成）→ 采购合同（履约跟踪）
```

> **注意**：物流入库单确认后，通知仓储库管创建业务入库单，库存正式形成并开始批次成本计算。

---

### 三、物流出库单（销售发货确认）

#### 3.1 核心场景

**场景一：常规销售出库**
```
销售提单 → 物流出库单 → 过磅出库 → 库存扣减
```

**场景二：以销定采直拨**
```
供应商直接发货给客户（不经过我方仓库）
→ 系统自动反向生成物流入库单（补完入库流程）
→ 下游确认收货后关联上游合同完成入库，再完成出库
```

**场景三：退货处理**
```
支持销售退货和采购退货的逆向物流流程
```

### 五、运费核算与承运商结算
**优先级：P1**

| 功能 | 说明 |
|------|------|
| 运费核算 | 按运价协议（按车次/按吨/按吨公里）核算运费 |
| 运费归集 | 按公司/部门拆分归集运费，喂给成本核价 |
| 承运商对账 | 汇总当期运单，与承运商对账运费明细 |
| 结算单生成 | 对账完成后生成承运商运费结算单 |

> **重要**：物流结算（运费，结给承运商）与采销结算（货款，结给供应商/收自客户）完全分离，分别走独立的结算流程。

---

### 六、在途监控
**优先级：P1**

| 功能 | 说明 |
|------|------|
| 轨迹跟踪 | 对接第三方物流平台，获取GPS/北斗实时位置 |
| 电子围栏 | 车辆进入厂区围栏时，自动推送"待收货预警"给现场人员 |
| 超时预警 | 超过预设时间未到达时触发告警 |
| 偏航预警 | 偏离既定路线超过阈值时触发告警 |
| 异常停留 | 非计划停留超时触发预警 |

> **一期说明**：在途监控依赖对接第三方物流平台，由物流供应商提供轨迹数据支持。

---

### 七、物流报表
**优先级：P2**

每月自动生成以下报表供财务使用：
- 运输台账（车次、里程、运费、签收状态一览）
- 运费汇总（按承运商/路线/期间汇总）
- 承运商业绩（准时率、异常次数等）

## 完整操作流程

**采购入库运输标准流程**：

```
① 接收采销系统的供应商发货指令
② 创建物流运单（关联发货指令）
③ 叫车派车（选择承运商/车辆/司机）
④ 在途跟踪（监控轨迹；异常预警）
⑤ 车辆到厂（电子围栏触发预警通知现场）
⑥ 地磅过磅（现场一线操作，数据回流物流运单）
⑦ 创建物流入库单（关联运单+磅单，确认到货数量）
⑧ 通知仓储库管创建业务入库单（库存正式形成）
⑨ 运输完成 → 运费核算 → 与承运商对账 → 生成运费结算单
```

**销售出库运输标准流程**：

```
① 接收采销系统的销售提单/发货通知单
② 创建物流出库运单（关联提单）
③ 叫车派车
④ 仓储库管备货/拣货
⑤ 过磅出库（地磅数据确认出库数量）
⑥ 创建物流出库单（库存扣减）
⑦ 客户签收 → 回单归档
```

## 下游任务流：你的操作触发

| 下游角色 | 触发事项 | 说明 |
|---------|---------|------|
| 现场一线（地磅） | 车辆进厂 | 电子围栏推送预警，现场准备过磅 |
| 仓储库管 | 物流入库单确认 | 通知库管创建业务入库单，批次进库存 |
| 仓储库管 | 物流出库单确认 | 库存扣减，出库完成 |
| 财务结算 | 运费结算单 | 运费数据归集到成本核价 |

## 模块关联关系

| 关联模块 | 关系说明 |
|---------|---------|
| 采销系统 | 接收发货指令/提单；运费回流成本 |
| 库存管理 | 到货触发入库；发货触发出库 |
| 地磅管理 | 进出场过磅数据关联运单/车辆，是数量确认依据 |
| 监控录像 | 到货/卸货影像关联车次 |
| 主数据管理 | 车辆、承运商、路线取自主数据 |
| 财务结算 | 运费、里程喂给成本核价和损耗评估 |
