# 矿产供应链运营管理系统
## 03 物流运营 · 物流管理

---

## 角色定位

物流运营是**货物从供应商到仓库、从仓库到客户的组织执行方**，负责叫车调度、运输任务全程跟踪、入库与出库确认，以及与承运商的磅单核对、亏吨核算与运费结算。运营平台方是物流的组织方，承运商是被调度和履约的一方。

> **典型岗位**：物流对接、物流台账核对、采购对接（如蔡宇豪、李纯福）

---

## 业务链位置

```
【采销业务员 · 合同管理】
（下达供应商发货指令 / 销售提单）
          ↓
【物流运营 · 物流管理】← 你在这里
     ↙              ↘
地磅管理            仓储库管
（车辆过磅）        （触发入库/出库）
     ↓
财务结算
（运费核算归集）
```

### 一、物流运单管理

#### 1.1 创建运单

| 创建方式 | 说明 |
|---------|------|
| 由供应商发货指令发起 | 采购入库场景，关联采购合同发货指令 |
| 由销售提单发起 | 销售发货场景，关联销售合同提单 |
| 独立创建 | 可独立创建后手动关联采购提货或销售提单 |

支持增删改查；关键操作需短流程审核并抄送相关人员。

#### 1.2 物流运单字段

**承运商信息**

| 字段名称 | 类型 | 说明 |
|---------|------|------|
| 承运商 | 下拉选择 | 从主数据选取，含资质和合同信息 |
| 车牌号 | 文本 | 执行本次运输的车辆 |
| 司机 | 文本 | 司机姓名 |
| 联系电话 | 文本 | 司机手机号 |

**运费信息**

| 字段名称 | 类型 | 说明 |
|---------|------|------|
| 运费承担方 | 单选 | 我方承担 / 供应商承担；选供应商承担时吨单价与总运费自动置灰 |
| 吨单价 | 数字（保留2位） | 元/吨，仅"我方承担"时可填 |
| 结算依据 | 下拉选择 | 按最小值（默认）/ 按我方磅单 / 按发货磅单 / 按协议固定值 |
| 协议固定值 | 数字（吨） | 仅选"按协议固定值"时显示 |
| 总运费 | 只读自动计算 | = 吨单价 × 结算重量；选"协议固定值"时可即时计算，其余在关联磅单后计算 |

**运输基础信息**

| 字段分组 | 字段名称 | 说明 |
|---------|---------|------|
| 提货信息 | 提货地点/时间 | 货物提取的地点与计划时间 |
| 提货信息 | 发货指令/提单号 | 关联的业务单据编号 |
| 送达信息 | 送达地点/预计到达时间 | 货物送达目的地与预计时间 |
| 运输信息 | 运输方式 | 汽运/铁运/船运等 |
| 运输信息 | 运输状态 | 待发运 / 在途 / 已到达 / 已签收 |
| 商品信息 | 物料/数量 | 运输的货物明细 |

#### 1.3 物流查询

对接第三方物流平台，查询货物在途实时状态。在第三方物流系统中获取当前位置、预计到达时间、在途异常信息。

### 二、关联磅单（手动匹配）
**优先级：P0**

运单详情页提供「关联磅单」功能，支持将三方磅单与运单手动关联，作为结算重量计算和亏吨核算的数据依据。

#### 2.1 磅单类型

| 类型 | 含义 | 必须性 |
|------|------|--------|
| 我方入库磅单 | 货物到达我方堆场时的过磅记录 | **结算必须**，缺失则无法提交结算 |
| 供应商发货磅单 | 供应商在发货时的过磅记录 | 可选，缺失时弹窗提醒可跳过 |
| 客户收货磅单 | 客户收货时的过磅记录（销售发货场景） | 可选，缺失时弹窗提醒可跳过 |

#### 2.2 操作流程

```
运单详情 → 点击「+ 关联磅单」→ 选择磅单类型 → 勾选对应磅单 → 确认关联
```

- 同一类型支持关联一条磅单
- 已关联磅单可随时移除
- 关联后自动更新结算重量与总运费显示

---

### 三、结算重量逻辑

根据运单的「结算依据」字段，系统自动计算结算重量：

| 结算依据 | 计算规则 | 备注 |
|---------|---------|------|
| 按最小值（默认） | min(我方入库重量, 供应商发货重量) | 两方均有磅单时取较小值；只有一方时取该方数值 |
| 按我方磅单 | 取我方入库磅单净重 | 需已关联我方磅单 |
| 按发货磅单 | 取供应商发货磅单净重 | 需已关联供应商磅单 |
| 按协议固定值 | 取填写的协议固定吨数 | 无需磅单即可计算 |

> **总运费** = 吨单价 × 结算重量（运费承担方为"我方承担"时计算，供应商承担时不计）

---

### 四、磅单对比与亏吨核算
**优先级：P1**

运单详情中展示三方重量对比与亏吨信息：

| 展示项 | 计算说明 |
|--------|---------|
| 我方入库磅单(吨) | 直接取关联磅单净重 |
| 供应商发货磅单(吨) | 直接取关联磅单净重 |
| 客户收货磅单(吨) | 直接取关联磅单净重 |
| 结算重量(吨) | 根据结算依据计算（见上节） |
| 亏吨数(吨) | max(我方, 供应商) − 结算重量；＞0 时以红色醒目展示 |
| 亏吨扣款额(元) | 亏吨数 × 吨单价；运费供应商承担时不计 |

---

### 五、运单结算校验

点击「提交结算」时，系统执行以下校验：

| 校验项 | 类型 | 处理方式 |
|--------|------|---------|
| 已关联我方入库磅单 | **强制** | 缺失则阻止结算，提示错误 |
| 已关联供应商发货磅单 | 警告 | 弹窗提醒，可选择继续 |
| 已关联客户收货磅单 | 警告 | 弹窗提醒，可选择继续 |
| 结算重量可计算 | **强制** | 根据结算依据无法得出有效数值时阻止结算 |

结算通过后，运单状态变更为「已结算」，生成结算单供财务处理。

### 六、物流入库单（采购到货确认）

#### 6.1 功能说明

货物过磅进入仓库后，通过物流入库单完成到货确认，触发库存形成流程。

**数据来源**：发货指令 + 物流运单 + 地磅过磅数据

#### 6.2 操作流程

```
方式一：选择发货指令 → 选择物流运单 → 地磅数据自动带入 → 入库确认
方式二：直接选择物流运单 → 手动关联地磅数据 → 入库确认
```

#### 6.3 关联关系

```
物流入库单 → 业务入库单（库存形成）→ 采购合同（履约跟踪）
```

> **注意**：物流入库单确认后，通知仓储库管创建业务入库单，库存正式形成并开始批次成本计算。

---

### 七、物流出库单（销售发货确认）

#### 7.1 核心场景

**场景一：常规销售出库**
```
销售提单 → 物流出库单 → 过磅出库 → 库存扣减
```

**场景二：以销定采直拨**
```
供应商直接发货给客户（不经过我方仓库）
→ 系统自动反向生成物流入库单（补完入库流程）
→ 下游确认收货后关联上游合同完成入库，再完成出库
```

**场景三：退货处理**
```
支持销售退货和采购退货的逆向物流流程
```

### 九、物流对账
**优先级：P1**

物流对账是独立页面，与调度看板、运单管理平级，集中处理磅单核重与承运商运费结算两类事务。

#### 9.1 页面结构

**Tab 一：承运商汇总**

按承运商维度汇总当期运费结算情况：

| 展示字段 | 说明 |
|---------|------|
| 承运商 | 承运方名称 |
| 当期车次 | 本期运输车次数 |
| 合计吨数 | 本期运输总量（吨） |
| 总里程(km) | 本期总里程 |
| 应付运费 | 按吨单价×结算重量合计 |
| 已结算 | 已完成结算的金额 |
| 亏吨扣款 | 本期亏吨扣款合计 |
| 实付金额 | 应付运费 − 亏吨扣款 |
| 结算状态 | 已结算 / 部分结算 / 待核算 |

支持「发起对账」（跳转到运单明细并筛选该承运商）和「查看结算单」操作。

**Tab 二：运单明细**

以单条运单为粒度展示对账明细，支持按承运商、结算状态、运费承担方、时间段筛选：

| 展示字段 | 说明 |
|---------|------|
| 运单号 | 可点击跳转运单详情 |
| 运输方向 | 采购入库 / 销售发货 |
| 承运商 / 司机 | 本次承运信息 |
| 吨单价 | 元/吨 |
| 结算依据 | 按最小值/我方磅单/发货磅单/协议固定值 |
| 我方磅单(吨) | 我方入库过磅净重 |
| 发货磅单(吨) | 供应商发货过磅净重 |
| 结算重量(吨) | 根据结算依据计算 |
| 亏吨(吨) | max重量 − 结算重量 |
| 亏吨扣款 | 亏吨 × 吨单价 |
| 总运费 | 吨单价 × 结算重量 |
| 承担方 | 我方承担 / 供应商承担 |
| 结算状态 | 已结算 / 待结算 / 缺磅单 |

#### 9.2 与运单管理的分工

| 模块 | 职责 |
|------|------|
| 运单管理 | 运单创建、在途跟踪、磅单关联、提交单条结算 |
| 物流对账 | 多运单汇总查看、按承运商对账、批量生成结算单、导出对账表 |

## 完整操作流程

**采购入库运输标准流程**：

```
① 接收采销系统的供应商发货指令
② 创建物流运单（关联发货指令；填写承运商、运费承担方、吨单价、结算依据）
③ 叫车派车（选择承运商/车辆/司机）
④ 在途跟踪（监控轨迹；异常预警）
⑤ 车辆到厂（电子围栏触发预警通知现场）
⑥ 地磅过磅（现场一线操作，数据回流物流运单）
⑦ 在运单详情中关联磅单（我方入库磅单为必须项；可补充供应商发货磅单）
⑧ 核查结算重量与亏吨信息
⑨ 提交结算（通过校验后状态变更为「已结算」）
⑩ 在物流对账页批量生成结算单 → 通知财务审核付款
⑪ 创建物流入库单 → 通知仓储库管创建业务入库单（库存正式形成）
```

**销售出库运输标准流程**：

```
① 接收采销系统的销售提单/发货通知单
② 创建物流出库运单（关联提单；填写运费信息）
③ 叫车派车
④ 仓储库管备货/拣货
⑤ 过磅出库（地磅数据确认出库数量）
⑥ 在运单详情中关联磅单（客户收货磅单可在对方确认后补录）
⑦ 创建物流出库单（库存扣减）
⑧ 提交结算 → 物流对账归集运费
⑨ 客户签收 → 回单归档
```

## 下游任务流：你的操作触发

| 下游角色 | 触发事项 | 说明 |
|---------|---------|------|
| 现场一线（地磅） | 车辆进厂 | 电子围栏推送预警，现场准备过磅 |
| 仓储库管 | 物流入库单确认 | 通知库管创建业务入库单，批次进库存 |
| 仓储库管 | 物流出库单确认 | 库存扣减，出库完成 |
| 财务结算 | 运费结算单 | 运费数据归集到成本核价；亏吨扣款体现在实付金额中 |
